### **三. 遍历逻辑**

- **步骤**：
    1. 启动 App（Airtest 连接）。
    2. 截图（snapshot()），截顶层区域，记指纹到界面集合。
    3. YOLO 实时识别 8 类按钮和参考信息，记组件和按钮总数。
    4. 操作（按配置优先级）：
        - **固定场景模式**：
            - 读 JSON（含类别 + 名称 + 上下文/ID + 预期 + 多屏标志）。
            - YOLO 识别目标按钮（结合参考项和指纹匹配上下文/ID），取边界框中心点击。
            - **多屏界面（如主页）**：没找到目标，滑动找（最多 5 屏），重新识别。
            - **非多屏界面**：没找到目标，报错或跳过，不滑动。
            - 操作后截图，断言指纹变或 YOLO 认预期元素。
        - **UI 深度遍历模式**：
            - 自动探索，按优先级：系统 > 注册 > 引导-拖动 > 引导-提示 > 操作 > 导航 > 提示 > 装饰。
            - 主界面滑动：swipe(600, 300, 100, 300) 或反之。
            - YOLO 识别顶层按钮，点击边界框中心。
            - 动态记录组件，操作后截图，断言指纹变。
            - 最大迭代 1000。
        - **路径全量遍历模式**：
            - 人为设"结束点"（比如"副本A"指纹）。
            - 用 BFS 找最短路径，DFS 筛选所有路径到"结束点"。
            - YOLO 识别按钮，点击实时坐标。
            - 滑动找目标（最多 5 屏）。
            - 记录路径，去重。
        - 点击：touch(x, y)（x, y 从 YOLO 动态计算）。
        - 拖动：swipe(x1, y1, x2, y2)。
    5. 截图，记录跳转。
    6. 重复。
- **记录**：用顶层指纹，主界面分屏记"主界面-屏X"，统计组件和按钮。

### **四. 结果分析**

- **可视化**：红框标未点，前后对比。
- **覆盖率**：
    - 界面：探索数/总数。
    - 组件：识别数/总数。
    - 按钮：点击数/总数。
    - 去重按钮：点击数/去重总数。
- **报告**：
    - JSON 包含：
        - 场景结果 + 断言 + 路径（含最短）。
        - 覆盖 UNSUREPORT：界面、组件、按钮、去重按钮的总数和覆盖率。

### **六、注意事项**

1. 多屏：滑动找目标，最多 5 屏，仅限特定界面（比如主页）。
2. 匹配：用参考信息（靠近/唯一/稳定），可选多参考。
3. 断言：操作前后对比顶层指纹或 YOLO 认预期。
4. OCR：用 PaddleOCR + 预处理，备用参考信息。
5. 图层：只点顶层，指纹算顶层。
6. 路径：BFS 找最短，DFS 盖所有。
7. 覆盖：动态统计总数和覆盖率。
8. 点击：YOLO 实时识别坐标，不写死。
9. 脚本：加上下文或 ID，提升可读性。
